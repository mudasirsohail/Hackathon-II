# Python Console Application Constitution

## Core Principles

### I. In-Memory Architecture
Data must be stored only in memory (lists, dicts, classes); No file system usage; Follow clean architecture principles

### II. Pure Python Implementation
No external libraries; Code must be readable, modular, and beginner-friendly; Use Python 3.10+ features only if necessary

### III. Spec-Driven Development
Every feature must be justified by the specification; Follow Spec-Driven Development strictly

### IV. Console Interface
Console output must exactly match the defined menu format; Build an in-memory Python console application

### V. Error Handling
No silent failures; Clear error messages for invalid input; No shortcuts

### VI. Clean Implementation
No overengineering; No missing menu options; Follow clean architecture principles
